# 主要思想

dsuontreedsu\ on\ treedsu on tree 又名树上启发式合并

（其实是非常暴力的一个东西。）

# 用途

可处理树上的一些统计类型的问题。

如：

  1. 求子树中颜色最多的颜色权值之和，相同数量都要加上 CF600E Lomsat gelral

  2. 给你一片森林，每次询问一个点与多少个点拥有共同的 KKK 级祖先 CF208E Blood Cousins

  3. 给一棵树，树上每个节点都有一个颜色，求已点 uuu 为根的子树中，出现次数 >=k>= k>=k 的颜色有多少种 CF570D Tree Requests

  4. ..................

# 过程

先把处理所有已 uuu 为根的所有轻儿子，并统计答案。

然后删除这些轻儿子的贡献，再把重儿子加进去。

然后不用删除，直接向上传到父亲节点，也就是 uuu 节点即可。

怎么样，是不是听着感觉很暴力，但由于我们最后重儿子是直接继承上去的，所以复杂度可以控制在 logloglog 内（跟树链剖分差不多）。

# 核心代码

# dfs

用来预处理出子树大小，重儿子的信息。

与树链剖分中的代码一模一样，不贴了。

# update

用于添加新的信息，或是删除不需要的信息。

# solve

这个是树上启发式合并的核心部分。

添加和删除操作类似于莫队，可以根据下面例题来理解。
