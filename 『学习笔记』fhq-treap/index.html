# 浅谈 fhq-treap（无旋 treap）

模板题：洛谷 P6136 【模板】普通平衡树（数据加强版）

（emm 刚写了这个，就放这个吧）

fhq−treapfhq-treapfhq−treap 也叫做 无旋 treap，由范浩强大佬发明。

个人认为是平衡树中码量最少，也最容易理解的一种写法。

# 主要思想

顾名思义，无旋意味着它没有旋转操作（终于没有恶心人的旋转了）。

事实上，无旋 treap 只有两种操作。

一种是 splitsplitsplit（分裂），另一种是 mergemergemerge（合并）。

平衡树中的各种操作都可以用这两个函数来搞定。下面我们来一一分析。

# 常见操作

  • 插入一个整数 xxx

  • 删除一个整数 xxx（若有多个相同的数，只删除一个）。

  • 查询整数 xxx 的排名（排名定义为比当前数小的数的个数 +1）。

  • 查询排名为 xxx 的数（如果不存在，则认为是排名小于 xxx 的最大数。保证 xxx 不会超过当前数据结构中数的总数）。

  • 求 xxx 的前驱（前驱定义为小于 xxx，且最大的数）。

  • 求 xxx 的后继（后继定义为大于 xxx，且最小的数）。

当然区间反转也是可以用无旋treaptreaptreap 实现的，这里先不讲。

# 函数解释

# split（分裂）

splitsplitsplit 函数要实现把一棵平衡树拆成两棵，左边的树上节点的值都小于等于 kkk, 右边的树上节点的值都大于等于 kkk。

  • x:x:x: 当前子树根节点

  • k:k:k: 以 kkk 为分界线分割树

  • a,b:a, b:a,b: 分裂之后两棵子树的根，左边的子树根是 aaa，右边的根是 bbb，这里要传地址，方便写。

注意： 当根为 0 时，一定一定一定要清空 aaa，&b&，不然会死循环（我就在这里卡了好久 QWQ）

# merge（合并）

mergemergemerge 函数要实现把两个分开的树再合并到一起去，并返回合并后的根节点。

  • x,y:x, y:x,y: 两棵树的根

这个操作类似于线段树合并。

注意：合并时只能是两个相邻的子树合并，不能跳着合并。

例如：树 rtrtrt ---> 子树aaa 和 子树bbb

子树bbb ---> 子树ccc 和 子树ddd （---> 表示拆成两棵树）

那么我们合并时，只能 merge(a,merge(c,d))merge(a, merge(c,d))merge(a,merge(c,d)) 或 merge(merge(a,c),d)merge(merge(a, c), d)merge(merge(a,c),d)。

而不能 merge(merge(a,d),c)merge(merge(a, d), c)merge(merge(a,d),c)

# insert（插入）

先新建一个点设为 yyy，把原树拆成 xxx 和 zzz。

xxx 中节点值 <=k<= k<=k，zzz 中节点值 >k>k>k。

然后一次合并 xxx，yyy，zzz 即可。

# remove（删除）

这个就很巧妙了，我们把值 <=k<=k<=k 的树先拆出来设为 aaa，然后把值 <k<k<k 的点再拆出来，剩余的部分设为为 ccc。

那么现在 ccc 中的点就是 =k=k=k 的。

我们直接合并 ccc 的左右两棵子树，这样就相当于把根节点删了。

最后合并回去。

# check_rk

查询 kkk 的排名。

我们把 <k<k<k 的的点从树上分裂出来，然后分出来的树的大小 +1 就是排名。

别忘了合并回去。

# check_val

查询排名为 kkk 的数。

我这里用的递归写法。

  • xxx 当前子树的根。

  • kkk 在当前子树中排名多少。

# check_pre

查询前驱。

这个也好说，我们先查排名，设排名为 rkrkrk，我们再差排名为 rk−1rk - 1rk−1 的数是多少，就是前驱了。

# check_next

查询 xxx 的后继，基本同理，但略有不同。

因为 xxx 可能有多个，我们查 xxx 的排名后 +1，不一定是下一个数，有可能还是 xxx。

那我们怎么办呢？

其实也不难，我们直接查 x+1x + 1x+1 的排名，没有？没关系，就算没有 x+1x + 1x+1，查出来的也是大于 xxx 的最小的数的排名。

然后我们再查一下这个排名的值即可。
