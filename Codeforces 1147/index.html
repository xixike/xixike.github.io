# A. Hide and Seek

看了好久没看懂题 /kk

后来意识到应该是只能移动到代币初始所在单元格的两旁的单元格，而不是当前所在单元格的两旁。

然后这题就简单很多了。

观察样例解释，发现只要起点或者终点有一个不一样就算不同方案。

换句话说，我们只考虑移动一步的情况，即向左，向右或者不动。

不难证明，这样所有的起点终点情况都会被考虑到。

考虑枚举每一个单元格，判断当前单元格能否作为：

  • 起点以及终点。
  • 起点在左边时的终点。
  • 起点在右边时的终点。

如何判断呢？

我们记录一下每个点被询问到的最小时间和最晚时间。

  • 如果代币在当前点一直不动，那么只有询问中没有出现当前点时合法，即：mni>mximn_i > mx_imni​>mxi​
  • 如果 mni>mxi−1mn_i > mx_{i - 1}mni​>mxi−1​，那么 iii 可以作为起点为 i−1i - 1i−1 时的终点。
  • 同理，如果 mni>mxi+1mn_i > mx_{i + 1}mni​>mxi+1​，那么 iii 可以作为起点为 i+1i + 1i+1 时的终点。

然后记录一下答案即可。

# B. Chladni Figure

一个比较显然的结论：

如果旋转 xxx 个点可以和原图形重合，那么旋转 k×xk \times xk×x 个点也可以和原图形重合。

显然旋转一圈（即旋转 nnn 个点）可以使得图形和原图形重合，因此我们只需要枚举 nnn 的约数并旋转即可。

判断是否重合只需要判断每条线段旋转之后的位置是否有线段即可。

线段的位置开个 pair\text{pair}pair 记录一下就行。

# C. Thanos Nim

博弈题。

先上结论：

nnn 堆石子中最小值出现次数小于等于 n2\frac n22n​ 时，先手胜；反之，先手败。

证明：

如果开始时最小值出现次数小于等于 n2\frac n22n​，那么先手可以选择其他 n2\frac n22n​ 堆不是最小值的石堆，从中取出若干石头使得它们变成最小值。

然后最小值出现次数大于 n2\frac n22n​ ，并且轮到后手。

后手最多只能选择 n2\frac n22n​ 堆石堆，使得它们成为最小值，然后就又轮到先手。

再考虑一下什么情况下游戏结束。

最小值为 0，且 0 的个数大于 n2\frac n22n​。

再根据上面所说的过程，后手操作完最多产生 n2\frac n22n​ 个 0，然后先手再操作即可获得胜利。

如果开始时最小值出现次数大于 n2\frac n22n​，那么先手选择的石堆中必定包括最小石堆，导致最小值减小，从而最小值出现次数必然小于等于 n2\frac n22n​，然后后手必胜。

所以结论成立。

# D. Palindrome XOR

题目中有个很奇怪的性质：

sss 的首个字符为 1。

又因为 a<ba < ba<b，可得 bbb 的第一位为 1，aaa 的第一位为 0。

因此我们 bbb 的最高位已经确定了，aaa 的最高位还没有确定，考虑先枚举 aaa 的最高位是第几位。

然后 aaa 和 bbb 的最高位是第几位都确定了，其他位如何处理呢？

考虑转化为图论模型。

现在的问题是：有 2n2n2n 个点分为两组，每组 nnn 个点，每个点都有一个点权为 000 或 111，有一些点的点权已经确定，有的点权还没确定。有一些限制条件形如 uuu 和 vvv 权值必须相同，或必须不同，求合法的不同点权方案数。

建一张有 2n+22n + 22n+2 个点的图，其中有两个点 0,10, 10,1，点权分别为 0,10, 10,1，如果有 u,vu, vu,v 两个点权值相同，那么 u,vu, vu,v 之间连权值为 000 的边，反之连权值为 111 的边。

具体来说：

  • 000 先向 111 连权值为 111 的边，表示 0,10, 10,1 点权不同。
  • 根据 sis_isi​ 连边：
    • 若 si=1s_i = 1si​=1，表示 ai,bia_i, b_iai​,bi​ 点权不同，ai,bia_i, b_iai​,bi​ 之间连权值为 111 的边。
    • 若 si=0s_i = 0si​=0 同理，令 ai,bia_i, b_iai​,bi​ 之间连权值为 000 的边。
    • 若 si=?s_i = ?si​=?，那么不连边。
  • 若 uuu 点权固定，那么向它的点权连权值为 000 的边。
  • 对于回文要求：bib_ibi​ 向 bn−i+1b_{n - i + 1}bn−i+1​ 连权值为 000 的边，表示 bib_ibi​ 和 bn−i+1b_{n - i + 1}bn−i+1​ 权值相同，aaa 同理。

建好图之后，我们可以把边权为 000 的边缩起来，这一步用并查集实现，然后就只剩下了边权为 111 的边。

对于每个联通块，显然都必须是二分图，否则不合法，判二分图直接黑白染色即可。

最后的贡献就是 2cnt−12^{cnt - 1}2cnt−1，cntcntcnt 为合法联通块个数，减 111 是因为 0,10, 10,1 那两个点所在联通块的权值已经固定了，没有贡献。

# E. Rainbow Coins

观察到最多 777 次询问，那跟序列长度肯定是没什么关系了。

我们肯定尽可能的把每次询问都问满，那么比较显然地询问：

  1. [a1,a2],[a3,a4]⋯[a2k−1,a2k][a_1, a_2], [a_3, a_4]\cdots[a_{2k - 1}, a_{2k}][a1​,a2​],[a3​,a4​]⋯[a2k−1​,a2k​]
  2. [a2,a3],[a4,a5]⋯[a2k,a2k+1][a_2, a_3], [a_4, a_5]\cdots[a_{2k}, a_{2k + 1}][a2​,a3​],[a4​,a5​]⋯[a2k​,a2k+1​]

然后相邻的两个硬币的颜色关系就已经知道了，我们再把颜色相同的合并一下，合并之后的颜色块为 b1,b2⋯btotb_1, b_2 \cdots b_{tot}b1​,b2​⋯btot​。

现在相邻两个 bbb 的颜色不同。

继续来询问：

  1. [b1,b3],[b2,b4],[b5,b7],[b6,b8]⋯[b_1, b_3], [b_2, b_4], [b_5, b_7], [b_6, b_8] \cdots[b1​,b3​],[b2​,b4​],[b5​,b7​],[b6​,b8​]⋯
  2. [b3,b5],[b4,b6],[b7,b9],[b8,b10]⋯[b_3, b_5], [b_4, b_6], [b_7, b_9], [b_8, b_{10}] \cdots[b3​,b5​],[b4​,b6​],[b7​,b9​],[b8​,b10​]⋯

此时我们就已经可以推出所有的颜色了，具体来说，设三种颜色分别为 1,2,31, 2, 31,2,3：

  • 设 b1=1,b2=2b_1 = 1, b_2 = 2b1​=1,b2​=2。
  • 如果 b1≠b3b_1 \neq b_3b1​=b3​，那么 b3=6−b1−b2b_3 = 6 - b_1 - b_2b3​=6−b1​−b2​，反之 b3=b1b_3 = b_1b3​=b1​。
  • b4b_4b4​ 同理。
  • 此时我们知道了 [1∼4],[5∼8]⋯[1 \sim 4], [5 \sim 8] \cdots[1∼4],[5∼8]⋯ 这样 4 个一组之内的颜色关系。
  • 第 4 次询问后我们又把所有组连在了一起：
  • 如果 b3≠b5b_3 \neq b_5b3​=b5​，那么 b5=6−b4−b3b_5 = 6 - b_4 - b_3b5​=6−b4​−b3​，反之 b5=b3b_5 = b_3b5​=b3​。
  • b6b_6b6​ 同理。
  • 然后我们就把 1∼41 \sim 41∼4 和 5∼85 \sim 85∼8 连起来了，后面的依次推过去即可。
