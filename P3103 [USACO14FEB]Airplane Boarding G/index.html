# Solution

刚开始看完题的时候感觉这只能暴力模拟……

于是不要脸的看了一眼题解之后，我们可以用一些表达式来表示每头奶牛到达它应该到的位置的时间，从而计算出总共时间。

具体来说，第 nnn 头牛到达其位置所用时间是 sn+tns_n + t_nsn​+tn​，第 n−1n - 1n−1 头牛到达 sns_nsn​ 的时间就是 sn+tn+1s_n + t_n + 1sn​+tn​+1（因为 cown−1cow_{n - 1}cown−1​ 初始坐标比 cowncow_ncown​ 的坐标靠左 1）

所以我们可以用二元组 (posi,timei)(pos_i, time_i)(posi​,timei​) 来表示一头牛到达 posipos_iposi​ 所花的时间为 timeitime_itimei​，对于一头牛有许多这样的二元组来限制它。

那么这只牛走到目的地的时间就是 max⁡{timei+(S−posi)}+T\max\{time_i + (S - pos_i)\} + Tmax{timei​+(S−posi​)}+T，二元组满足条件 posi≤Spos_i \leq Sposi​≤S。

我们再把 SSS 提出来，所以要找的就是 timei−positime_i - pos_itimei​−posi​ 的最大值。

但是目前复杂度还是 n2n^2n2 的，我们还要考虑如何优化。

不难发现，对于两个二元组限制条件 (a,b)(a, b)(a,b) 和 (c,d)(c, d)(c,d)，如果 a<ca < ca<c 且 d−c<b−ad - c < b - ad−c<b−a，那么 (c,d)(c, d)(c,d) 对于以后所有的奶牛都是没有用的。

d−c<b−ad - c < b - ad−c<b−a 使得 (c,d)(c, d)(c,d) 对于当前的奶牛没有贡献，再加上 a<ca < ca<c 使得 (c,d)(c, d)(c,d) 这个条件对于后面所有的奶牛都没有了贡献。

所以我们直接把这个条件删掉即可。

用平衡树维护一下 pospospos 和 time−postime - postime−pos 的值，以及 time−postime - postime−pos 的最大值，倒序枚举每一头奶牛：

  • 查询：把 pos≤Sipos \leq S_ipos≤Si​ 的数切下来查询。
  • 修改：当前插入的二元组是 (a,b)(a, b)(a,b)，然后把树中 d−c>b−ad - c > b - ad−c>b−a 的点全都删掉。

然后就没别的了，具体见代码吧。
