# A. 跳蚤王国的宰相

一上来就冲了个换根 dp\text{dp}dp，然后过了小样例和自己手造的样例，感觉还挺稳的，考到一半发现题看假了，样例也算假了，想成要删多少个点了（我就说这题怎么这么水 QwQ）。

回到本题，首先重心的答案肯定是 0。

考虑重心的性质：如果一个点为重心，那么它的所有子树的大小都不大于 n2\frac n22n​，

所以我们只要把一个点为根的情况下它所有子树的大小全部减到 n2\frac n22n​ 以下即可。

详细过程：

  • 先把重心拎出来，把它的所有子树从大到小排序，并做个前缀和存到 sumisum_isumi​ 里。然后依次递归处理每一棵子树。

  • 对于每棵子树进行分类讨论，设 ididid 为子树的根，xxx 为每棵字数内的点：

    • 如果 ididid 本身就是重心（重心可能有多个），直接更新答案 ansx=(x !=id)ans_x = (x\ != id)ansx​=(x !=id)，如果当前的 xxx 不是 ididid 只需要切一条边即可。
    • 如果减去 sumpos[x]−1sum_{pos[x] - 1}sumpos[x]−1​ 之后 xxx 可以成为重心，那么在 1∼pos[x]−11 \sim pos[x] - 11∼pos[x]−1 之间二分需要切掉多少条边（也就是切多少棵子树，然后把这些子树接到 xxx 下面）。
    • 如果减去之后 xxx 依然不能成为重心，那么就去 pos[x]+1∼totpos[x] + 1 \sim totpos[x]+1∼tot 里面二分找答案。

    各种边界什么的就不细讲了，比较繁琐，直接看代码吧。

# B. 事情的相似度

考场上完全没思路，暴力也懒得打了（其实是没时间）。

首先转换一下题意：建出后缀自动机，题目就是求 l∼rl \sim rl∼r 内两个串的最长公共后缀，就是所代表节点的 lcalcalca 的 lenlenlen。

把询问离线下来，对于每个 rrr 依次处理。

每加进去一个字符，我们把 SAM\text{SAM}SAM 中相应的点到根的路径覆盖上颜色 iii（iii 为字符编号），这个过程使用 LCT\text{LCT}LCT 维护。

然后用树状数组统计从根到每个点的最大值（这里由于是从根到每个点，所以在树状数组上是一段后缀，插入和查询的循环要反过来）。查询的时候直接查询就可以了。
