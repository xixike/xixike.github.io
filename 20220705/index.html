# B. 树上的僵尸

这道题就复杂多了，考虑 dp\text{dp}dp。

设 fxf_xfx​ 表示两个端点都在 xxx 子树内且经过 xxx 的合法路径数，gxg_xgx​ 表示一端在子树内另一端在子树外的合法路径数。

通过容斥思想，我们用总数减去不合法的来计算答案，不合法的即为所有路径都经过同一个点的情况。

直接枚举点肯定会超时，我们枚举所有路径经过深度最浅的点，那么不合法方案数就变为了总数减去不是最浅的方案数。

总数为 (fi+gi)m(f_i + g_i)^m(fi​+gi​)m，不是最浅的方案数为 gimg_i^mgim​，答案为：

ans=(∑i=1nfi)m−∑i=1n((fi+gi)m−gim)ans = (\sum_{i = 1}^nf_i)^m - \sum_{i = 1}^n((f_i + g_i) ^ m - g_i ^m) ans=(i=1∑n​fi​)m−i=1∑n​((fi​+gi​)m−gim​)

再来看 fif_ifi​ 和 gig_igi​ 如何计算：

  • n≤1000n \leq 1000n≤1000：暴力计算，转移的时候就是求编号小于当前点且距离当前点 kkk 以内的点的个数，用树状数组维护前缀和即可，复杂度 O(n2log⁡n)O(n^2\log n)O(n2logn)。

  • k=n−1k = n - 1k=n−1：fxf_xfx​ 为 xxx 子树内任意两点配对个数减去所有儿子自己配对个数，gxg_xgx​ 为 (n−sizx)×sizx(n - siz_x) \times siz_x(n−sizx​)×sizx​。

  • 链：因为是一条链，且要经过 xxx，相当于有一个端点必须在 xxx，fxf_xfx​ 为 min⁡{k+1,n−i+1}\min\{k + 1, n - i + 1 \}min{k+1,n−i+1}，gxg_xgx​ 为 xxx 向上 kkk 步内 fxf_xfx​ 的和减去 xxx 到 xxx 的 kkk 级祖先内两两配对的个数。

k=0k = 0k=0 的情况答案就是 nm−nn^m - nnm−n。

然后这题就解决了。

# C. 豌豆射手

先想一个比较暴力的做法，枚举豌豆射手的排列，然后尽可能压缩空间的放置。

假设占用的长度为 sss，考虑把后面 d−sd - sd−s 格的空位置插到前面 n+1n + 1n+1 个位置中，可以为空，方案数为 (d−s+nn)\dbinom {d - s + n}{n}(nd−s+n​)。

但是枚举排列显然是过不了的，考虑如何通过 dp\text{dp}dp 直接计算出占用 sss 的草坪时的方案数。
