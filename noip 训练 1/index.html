# CF402E Strictly Positive Matrix

# Description

洛谷传送门

# Solution

考虑矩阵乘法的本质：

ci,j=∑k=1n(ai,k×bk,j)c_{i,j} = \sum\limits_{k = 1}^{n}{(a_{i,k} \times b_{k,j})}ci,j​=k=1∑n​(ai,k​×bk,j​)

实际上类似于 FloydFloydFloyd 算法，枚举一个中间点 kkk，计算从 iii 到 jjj 的最短路，只不过把加法变成了乘法。求的不再是最短路了，但是我们依旧可以从 iii 向 jjj 连边。

我们可以先对输入的矩阵进行一些处理，把大于 0 的点全都赋值为 1。这样就类似于邻接矩阵了。

对于 gi,jg_{i,j}gi,j​ 如果大于 0，就表示有一条从 iii 到 jjj 的边，我们就给它连上，然后跑个 TarjanTarjanTarjan 缩点，看是不是只有一个强连通分量即可。

# CF464E The Classic Problem

# Description

洛谷传送门

# Solution

emm…… 这是我为数不多的黑题之一，所以来写篇博客记录一下。

我们发现边权过大，只能用高精度来算，但是这样的复杂度太劣了，无法通过此题。

观察到边权只能是 2x2^x2x，所以我们可以给它压成二进制数，然后跑最短路时单点加。

我们再来考虑一下 dijkstradijkstradijkstra 需要哪些操作：

  • 区间加

  • 比大小

先来看区间加，如果第 xxx 位为 0，那么赋值为 1 即可，如果第 xxx 位为一，那么我们需要向上找第一个为 0 的位，然后这一段区间赋值为 0，第一个为 0 的位赋值为 1。

这个查找操作可以使用线段树上二分来实现，据说暴力查找也是可以过的，数据比较水。

那区间赋值为 0 如何实现呢？主席树似乎不太能支持修改。我们可以建一棵全 0 主席树，然后修改时直接把这棵全 0 的树拍上去。

再来看区间比大小，直接比较区间和即可。

然后就是代码实现了，个人认为非常有助于提升码力，也让我对主席树的理解更深了一步。

# CF980E The Number Games

# Description

洛谷传送门

# Solution

向上面那道题一样的贪心，能选高位就选高位。

所以以 nnn 号选手为根，从 n−1n - 1n−1 依次向下枚举每一位选手，判断能不能留下。

如何判断呢？就是要判断从当前点到根节点要联通的话还需要增加多少个点。

如果小于还能增加的节点数，就可以：否则不行。

用树链剖分维护一下即可。

# CF1101F Trucks and Cities

Luogu 传送门

# Solution

看完题目后，首先想到的是二分，而然复杂度不是很优秀，并不能通过此题，但是 ywyywyywy 学长用玄学优化二分通过辣，太神啦 CCCCCCOrz。

我们要对题目进行转化：

如果有一辆卡车从 sss 行驶到 ttt，每行驶一格需要耗 的 ccc 油，最多可以加 次油kkk，那么成功行驶需要的油箱的最小值为：以城市为端点，划分为 k+1k + 1k+1 段，它们中长度最大的段的最小值，再乘上 就是油箱 ccc 的大小。最后对于每辆卡车答案取一个 maxmaxmax 就可以了。

我们设 dpi,j,kdp_{i, j, k}dpi,j,k​ 表示把第 iii 到 jjj 个城市划分为 kkk 段后，最大的一段的最小值。

那么转移方程就是：

dpi,j,k=min⁡l=ij(dpi,j,k,max(dpi,l,k−1,aj−al))dp_{i, j, k} = \min\limits_{l = i}^j(dp_{i, j, k}, max(dp_{i, l, k - 1}, a_j - a_l))dpi,j,k​=l=iminj​(dpi,j,k​,max(dpi,l,k−1​,aj​−al​))

emm…… 有点长，但这样的转移是 O(n4)O(n^4)O(n4) 的，还是无法通过此题。

考虑优化，不难发现 dpi,l,k−1dp_{i, l, k - 1}dpi,l,k−1​ 随 lll 增大而增大，aj−ala_j - a_laj​−al​ 随 lll 增大而减小，所以我们找中间的断点使得 dpi,l′,k−1<=aj−al′dp_{i, l', k - 1} <= a_j - a_{l'}dpi,l′,k−1​<=aj​−al′​ 且 dpi,l′+1,k−1>=aj−al′+1dp_{i, l' + 1, k - 1} >= a_j - a_{l' + 1}dpi,l′+1,k−1​>=aj​−al′+1​，然后转移即可，这样复杂度就变为了 O(n3)O(n^3)O(n3)。
