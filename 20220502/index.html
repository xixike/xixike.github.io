# A. gcd

考过原题。

一个非常显然的结论（假设 a>ba > ba>b ）：

gcd⁡(a,b)<=a−ba⊕b>=a−b\gcd(a, b) <= a - b\\ a \oplus b >= a - bgcd(a,b)<=a−ba⊕b>=a−b

那么我们先枚举一个数 iii 作为 gcd⁡\gcdgcd，然后枚举它的倍数 j(2×i≤j≤n)j\ (2 \times i \leq j \leq n)j (2×i≤j≤n)，判断 j⊕(j−i)j \oplus (j - i)j⊕(j−i) 是否等于 iii，是的话就让答案加 1。

# B. max

同样有原题（就是多了个多测）。

类似于 【HH 的项链】 这道题。

考场上的暴力：把每个数的因数全都预处理出来，然后每次查询 l∼rl \sim rl∼r 时，把 l∼rl \sim rl∼r 中所有数的因数全都放到桶里，答案就是最大的出现至少两次的数，再整个莫队优化一下。

复杂度我也不知道是多少，反正是过不了 QwQ

现在来说正解：

首先还是要把每个数的因数全都预处理出来，然后把询问按照右端点排序。

从左到右依次枚举每一个数 aia_iai​，然后枚举 aia_iai​ 的所有因数，判断这个因数之前是否出现过（出现过两次及以上就可以当作 gcd⁡\gcdgcd），如果出现过就把这个因数插到树状数组第 lastposlastposlastpos 位上，维护一个前缀最大值。

注意时刻更新 lastposlastposlastpos。

查询的时候枚举以当前点 iii 为右端点的询问，在树状数组上查询 q[i].lq[i].lq[i].l 为左端点的后缀最大值，就是答案了。

非常妙的树形背包。

设 fx,if_{x, i}fx,i​ 表示以 xxx 为根的子树内与 xxx 颜色相同的联通块大小为 iii 时合法的方案树，不考虑颜色。

sumx=∑i=1min⁡(l,siz[x])fx,i×(k−1)sum_x = \sum_{i = 1}^{\min(l, siz[x])} f_{x, i} \times (k - 1)sumx​=i=1∑min(l,siz[x])​fx,i​×(k−1)

乘 (k−1)(k - 1)(k−1) 是因为不能和父亲的颜色相同。

然后再来看 fx,if_{x, i}fx,i​ 如何转移。

分两种情况讨论：

  • xxx 的儿子 yyy 和 xxx 颜色相同，那么就是树上背包板题了，先枚举与 xxx 颜色相同的联通块大小，然后枚举 yyy 子树内与 yyy 颜色相同的联通块大小（此时 colx=colycol_x = col_ycolx​=coly​），那么有：

    fx,i=∑jfx,j×fy,i−jf_{x, i} = \sum_{j}f_{x, j} \times f_{y, i - j}fx,i​=j∑​fx,j​×fy,i−j​

  • 儿子 yyy 的颜色与 xxx 不相同，那就很简单了，直接令 fx,if_{x, i}fx,i​ 乘上 sumysum_ysumy​ 再加上刚才背包计算出来的结果就行。

    （至于为什么能直接乘 sumysum_ysumy​，我只能说 sumysum_ysumy​ 的定义实在是巧妙）
