# A. Garland

简单分讨。

考虑一段 0 的两端奇偶性不同，那么不管怎么放代价只会加 1.

所以只考虑连段奇偶性相同的。

如果数字数量足够填满当前段，那么没有代价，否则代价是 2（自己手模一下）。

所以把两端奇偶性相同的空段按长度从小到大排序，能填满就填满即可。

还有一些奇奇怪怪的小细节，自己写代码的时候慢慢调吧。

# B. Numbers on Tree

这道题似乎没有那么复杂。

大致思路是无脑 dfs\text{dfs}dfs 一遍求出每个点之间的相对大小关系，然后依次赋值即可。

首先如果出现 cic_ici​ 大于等于 iii 的子树大小，就无解。

然后 dfs 回溯过程暴力合并序列，最后把 xxx 移动到 cx+1c_x + 1cx​+1 的位置即可。

# C1. Madhouse (Easy version)

C1\text{C1}C1 较简单，你只有 3 次询问机会，不管怎么试都能试出来吧……

那么正确的询问方法是先问 [1,n][1, n][1,n] 再问 [2,n][2, n][2,n]。

然后你的第一次询问比第二次询问子串数多 nnn 个。

开个 map\text{map}map 记一下依次推出每一个字符即可。

总询问字符串数为 n(n+1)2+n(n−1)2\frac {n(n + 1)}{2} + \frac {n(n - 1)}{2}2n(n+1)​+2n(n−1)​。

# C2. Madhouse (Hard version)

现在你总询问子串数不能超过 0.777(n+1)20.777(n + 1)^20.777(n+1)2 了。怎么办呢？

假设你的字符串为 sss，现在你询问 [1,n][1, n][1,n]。

然后你考虑 sss 中的每一位 sis_isi​ 在不同长度子串中出现的次数。

经过一番观察不难发现，在所有不同长度的子串中出现次数全部相同的字符个数最多两个。（出现一次的是奇数长度时最中间的字符）

比如与 sxs_xsx​ 在所有长度子串出现次数完全相同的是 s2⌊n2⌋−xs_{2\lfloor\frac n2\rfloor - x}s2⌊2n​⌋−x​（总之就是轴对称）。

所以我们先询问 [1,n2][1, \frac n2][1,2n​] 和 [2,n2][2, \frac n2][2,2n​] 得到 1∼n21 \sim \frac n21∼2n​ 的答案。

然后询问 [1,n][1, n][1,n]，再根据上面说的对称性求出 [n2+1,n][\frac n2 + 1, n][2n​+1,n] 的答案。

总子串个数 2n2(n2+1)2+n(n+1)2≈0.75n22\frac {\frac n2(\frac n2 + 1)}{2} + \frac {n(n + 1)}{2} \approx 0.75n^2222n​(2n​+1)​+2n(n+1)​≈0.75n2。

# D. LCC

首先一个比较显然的结论是肯定是相邻两个点相撞，不解释了。

一共有三种碰撞情况：都向左，相向，都向右。

不难想到去枚举哪两个点相撞，假设是 iii 和 i+1i + 1i+1 相撞。

设 limi,j,klim_{i, j, k}limi,j,k​ 表示 iii 向 jjj 方向走，i+1i + 1i+1 向 kkk 方向走是被禁止的。

记 fi,0/1f_{i, 0/1}fi,0/1​ 表示考虑了前 iii 个点，当前点是向左 / 右走，且与 i+1i + 1i+1 的碰撞是最早的概率。

pip_ipi​ 是往左走的概率，qiq_iqi​ 反之。

转移方程:

fi,0=pi×(fi−1,0×[limi−1,0,0=0]+fi−1,1×[limi−1,1,0=0])fi,1=qi×(fi−1,0×[limi−1,0,1=0]+fi−1,1×[limi−1,1,1=0])f_{i, 0} = p_i \times (f_{i - 1, 0} \times [lim_{i - 1, 0, 0} = 0] + f_{i - 1, 1} \times [lim_{i - 1, 1, 0} = 0])\\ f_{i, 1} = q_i \times (f_{i - 1, 0} \times [lim_{i - 1, 0, 1} = 0] + f_{i - 1, 1} \times [lim_{i - 1, 1, 1} = 0])fi,0​=pi​×(fi−1,0​×[limi−1,0,0​=0]+fi−1,1​×[limi−1,1,0​=0])fi,1​=qi​×(fi−1,0​×[limi−1,0,1​=0]+fi−1,1​×[limi−1,1,1​=0])

每次修改 limlimlim 后都要把后缀再做一遍，复杂度是 O(n2)O(n^2)O(n2) 的，过不了。

如何优化呢？考虑线段树。

线段树上存 valrt,j,kval_{rt, j, k}valrt,j,k​ 表示 rtrtrt 对应区间 [l,r][l, r][l,r] 的 lll 向 jjj 走，rrr 向 kkk 走且当前枚举的 iii 和 i+1i + 1i+1 最先相撞的概率。

更新的话从当前点暴力往上跳暴力修改。

答案就是根节点 4 个值加起来。

iii 和 i+1i + 1i+1 碰撞的概率是前 i−1i - 1i−1 个不碰撞减去前 iii 个不碰撞。

# F. Harry The Potter

感觉这个比上面两道题稍微简单一点。

我们假设对 (i,j)(i, j)(i,j) 进行二操作，那么在 (i,j)(i, j)(i,j) 之间连一条边。

不难证明，最后形成的一定是森林，设连通块个数为 CCC，答案就是 n−Cn - Cn−C，问题变成了如何最大化 CCC。

我们再把这个问题分成两部分。

  • 判断一个点集 SSS 是可以由二操作连成一棵树的。

  • 跑一个枚举子集 dp\text{dp}dp 的东西来求值。

第二个部分是简单的，O(3n)O(3^n)O(3n) 即可。

下面只讨论第一个部分。

观察到二操作是对 iii 减 xxx，对 jjj 减 x+1x + 1x+1。这个 +1+1+1 有点恶心，我们先不考虑它。即对 i,ji, ji,j 都减去 xxx。

那么一个点集能否由二操作连起来的条件即为：把这个点集 SSS 分成两部分 S1,S2S1, S2S1,S2，使得这两部分的和 sums1=sums2sum_{s1} = sum_{s2}sums1​=sums2​。

现在把 +1+1+1 的条件考虑进去。

条件就变为了：∣sums1−sums2∣≤∣S∣−1|sum_{s1} - sum_{s2}| \leq |S| - 1∣sums1​−sums2​∣≤∣S∣−1.（∣S∣|S|∣S∣ 是点集大小）

所以折半搜一下所有的和，然后双指针跑一跑就行。

还有一个小问题。

设有两个数分别为 8,−98, -98,−9，它们分成两部分后差的绝对值最小为 ∣8+(−9)∣=1|8 + (-9)| = 1∣8+(−9)∣=1.

1 是小于 2 的，符合条件。

但是我们真的可以一步把它们两个都变成 0 吗？显然不行。

所以遇到这种所有数都在同一部分还符合条件的情况，要特判掉。
