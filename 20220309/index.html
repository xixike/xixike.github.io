# B. 素数

我们把每个数看做一个节点，和为素数的连边，然后求最大匹配，设为 kkk。当 m<km < km<k 时，答案就是 2×m2 \times m2×m；反之，设 sumsumsum 为所有可能的和其他牌凑出素数的牌数，答案为 min⁡(sum,k+m)\min(sum, k + m)min(sum,k+m)。

接下来看如何求最大匹配。

和为素数的两个数必定是一奇一偶（除非两个数都是 1）。

  • 先不考虑 1，我们把偶数放到左边，奇数放到右边，建二分图，使用 dinic\text{dinic}dinic 算法求最大匹配为 tmp1tmp1tmp1。

# C. 广播

暴力做法可以强行连边跑最短路。

  • 点分树

先建出点分树，然后在点分树上预处理出一个点可以到达哪些点，以及哪些点可以到达这个点，同时记录下消耗的距离，都存到 vector\text{vector}vector 里面。

由于预处理的过程是 bfs\text{bfs}bfs，所以消耗距离是从小到大的，这时我们把它 reverse\text{reverse}reverse 一下，为了后面方便删点。

然后跑一遍 bfs\text{bfs}bfs，设当前点为 xxx，枚举所有能到达 xxx 的点 yyy，计算一下还能监控的距离，然后枚举 yyy 能到达的点 vvv（倒序枚举），判断是否还能监控到，如果可以，更新答案，并把 vvv 弹出 vector\text{vector}vector，否则退出（因为倒序是从小到大的）。
