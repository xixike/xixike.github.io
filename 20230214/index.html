# A. 偷税计划

一直没有意识到大于 1 的数个数小于等于 25 的性质，剩下的都是 1。

思考一个 dp\text{dp}dp 来计算这个东西。

设 dpm,idp_{m, i}dpm,i​ 表示从 2∼m2 \sim m2∼m 中选择了 iii 个数，使得这 iii 个数互质的方案数。

答案就是:

ANS=∑i=0mdpm,i×(ni)×i!ANS = \sum_{i = 0}^mdp_{m, i}\times \dbinom ni \times i!ANS=i=0∑m​dpm,i​×(in​)×i!

那么就产生了 3 种做法来计算这个 dpm,idp_{m, i}dpm,i​：

  1. 暴力打表，设 fi,sf_{i, s}fi,s​ 表示考虑了前 iii 个数，当前质数状态是 sss 的方案数。空间可能比较大，但是没关系，本地能跑就行。

  2. 把第 16∼2516 \sim 2516∼25 个质数单独考虑，因为它们只能是独自出现，不能再乘上其他质数了，这样质数状态只用开到 2152^{15}215。

  3. 根号分治，还是设 fm,j,sf_{m, j, s}fm,j,s​ 表示从 2∼m2 \sim m2∼m 中选了 jjj 个数，质数状态为 sss 的方案数。

这里详细讲一下第三种做法。

我们把 2,3,5,72, 3, 5, 72,3,5,7 看作小质数，111111 以后的看作大质数，这样一来一个数只会包含一个大质数和若干小质数。

所以我们状态只用记 242^424，转移就类似背包。

# D. 基因进化

假设要翻转两个前缀分别是 i,j(j<i)i, j(j < i)i,j(j<i)。

那么 iii 的翻转是不影响 1∼j1 \sim j1∼j 的，也就是 1∼j1 \sim j1∼j 一定能取到最小字典序的情况。

所以我们开一个双端队列就记当前已经考虑过的位置的最小字典序串是什么样的。

加进来一个新串有两种情况：

  1. 正序插到队列后面。

  2. 倒序插到队列前面（注意这里原答案串是不需要翻转的，因为当前字典序是最小的，且也可以表示出来）

插进来后二分 + Hash 把字典序小的留下来，大的删掉。

实现就是要动态维护 Hash。
