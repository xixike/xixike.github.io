# A2. Gardener and the Capybaras (hard version)

AAA 比较简单，贪心找一段足够小或足够大的串当成 b\text{b}b 即可。

# B. Gardener and the Array

简单结论。

如果一个数字 cic_ici​ 包含的所有二进制位中有至少一位在所有数中只出现了一次，那么这个 cic_ici​ 就是必要的。

如果 nnn 个数中有至少一个数不是必要的，那么就是 'Yes'，否则是 'No'。

对于如何判断一个数是否是必要的，我们开个桶记录每个二进制数位出现了多少次。如果只出现了 1 次，那么包含这个二进制数位的数就是必要的。

# C. Interesting Sequence

简单题（感觉比 BBB 简单）.

按位考虑，稍微讨论一下可以得出从 nnn 变成 xxx，需要的各种限制，输出最小的即可。

# D. Friendly Spiders

把 3e53e53e5 以内的质数全都筛出来。

然后对于每个 aia_iai​ 分解质因数，从 aia_iai​ 向他们的质数连边，然后 bfs\text{bfs}bfs 一遍即可。

# E. The Human Equation

结论题（

设 bbb 是 aaa 的前缀和数组。

那么每次操作相当于在 bbb 中任选一些数，令它们 +1+1+1 或 −1-1−1。

所以最后的答案就是 mx−mnmx - mnmx−mn，初值都为 0。

所以我们先单独考虑一个角。

不难发现，删掉一个角后，每一行的格子数单调不降或单调不升的，这让我们想起了什么？

没错，就是整数划分。

然后我们现在有四个角要如何处理呢？
