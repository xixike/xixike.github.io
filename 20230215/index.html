# A. 取石子

看了几个小时才想到怎么博弈。

我们把 xi  mod  (a+b)x_i\ \bmod\ (a+b)xi​ mod (a+b) 的余数分类讨论，设余数为 rrr。

假设 a<ba < ba<b.

  1. 0≤r≤a−10 \leq r \leq a - 10≤r≤a−1

  2. a≤r≤b−1a \leq r \leq b - 1a≤r≤b−1

  3. b≤r≤2a−1b \leq r \leq 2a - 1b≤r≤2a−1

  4. 2a≤r≤a+b−12a \leq r \leq a + b - 12a≤r≤a+b−1

下面来分析一下：

  • 情况 1 对胜负没用，忽略掉，不过最后计数的时候要乘上 2c12^{c_1}2c1​。

  • 出现情况 2，那么 Alice\text{Alice}Alice 必胜，因为这步可以留到最后走。

  • 出现情况 3，那么相当于交换先后手。

  • 出现情况 4：

    • 出现次数大于 1：那么 Alice\text{Alice}Alice 必胜，可以手模一下。

    • 只出现 1 次，Alice\text{Alice}Alice 先手则先手胜，Bob\text{Bob}Bob 先手则交换先手顺序。

如果 2a<b2a < b2a<b 的话把 3，4合并起来就行了（好像）

然后把四种情况记到桶里。

O(1)O(1)O(1) 计算或者 O(n)O(n)O(n) dp\text{dp}dp 都行。

# C. 树

挺复杂的格路计数。

经过大量模拟不难发现不包含 kkk 连树相当于根到每个叶子上左儿子个数小于 kkk。

考虑按先序遍历建出这棵树，每次有两种操作：

  1. 当前点加一个左孩子；

  2. 退到上一个只有左孩子的点，加一个右孩子。

转化成括号匹配，相当于一个长度为 2n−22n - 22n−2 的序列，保证左括号 - 右括号数量小于 mmm。

当然还要保证括号序列合法，即前缀和大于等于 0。

如果只有前缀和大于等于 0 的条件，就是经典的卡特兰数的容斥：

总路径 - 不合法路径。大概就是把第一次碰到 -1 后面的路径全部翻折，最后到达 (2n−2,−2)(2n-2, -2)(2n−2,−2)（不会的自行百度）

不合法方案数就是从 (0,0)(0, 0)(0,0) 走到 (2n−2,−2)(2n-2,-2)(2n−2,−2) 的路径数。

然后现在有两个限制条件，我们的方案数就是：

总−[−1]−[m−1]总 - [-1] - [m - 1] 总−[−1]−[m−1]

（[−1],[m−1][-1], [m-1][−1],[m−1] 表示经过 −1,m−1-1, m-1−1,m−1 的方案数）

但是明显有重复的，所以就再套个容斥：

总−[−1]−[m−1]+[−1,m−1]+[m−1,−1]−[−1,m−1,−1]−⋯总 - [-1] - [m - 1] + [-1, m - 1] + [m - 1, -1] - [-1, m - 1, -1] -\cdots 总−[−1]−[m−1]+[−1,m−1]+[m−1,−1]−[−1,m−1,−1]−⋯

然后就没了。
