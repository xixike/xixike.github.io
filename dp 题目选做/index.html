# UVA1292 Strategic game

树形 dp\text{dp}dp 板子题。

直接设 fx,0/1f_{x, 0/1}fx,0/1​ 表示 xxx 子树内所有的边都被看守到，且 xxx 不放 或者 放 士兵的最少士兵数。

  • xxx 如果不放士兵，它的所有儿子节点都必须放士兵，即：

    dpx,0+=dpy,1dp_{x, 0} += dp_{y, 1}dpx,0​+=dpy,1​

  • xxx 如果放士兵，那么它的儿子可放可不放，即：

    dpx,1+=min⁡{dpy,0,dpy,1}dp_{x, 1} += \min\{dp_{y, 0}, dp_{y, 1} \}dpx,1​+=min{dpy,0​,dpy,1​}

# P7690 [CEOI2002] A decorative fence

非常神仙的 dp\text{dp}dp，不理解为什么只是绿牌题。

这题看到之后毫无思路 QwQ，可能是我太菜了于是看了一眼题解。

我们设 dpi,j,0/1dp_{i, j, 0/1}dpi,j,0/1​ 表示从左到右放了 iii 块木板，最左边的一块是第 jjj 低的，且 0 表示最左边的木板低于它右边的那个，1 反之时的方案数。

不难得到转移方程：

dpi,j,0=∑k=ji−1dpi−1,k,1dpi,j,1=∑k=1j−1dpi−1,k,0dp_{i, j, 0} = \sum_{k = j}^{i - 1} dp_{i - 1, k, 1} \\ dp_{i, j, 1} = \sum_{k = 1}^{j - 1}dp_{i - 1, k, 0}dpi,j,0​=k=j∑i−1​dpi−1,k,1​dpi,j,1​=k=1∑j−1​dpi−1,k,0​

就是把当前最左边的一位删掉之后可能的状态。时间复杂度 O(n3)O(n^3)O(n3)

dp\text{dp}dp 之后的输出方案更是困难，但是这里只讲 dp，不是题解，所以不说了。

dpa,b+1,c,d,e+=dpa,b,c,d,e(b<n2,a>b)dp_{a, b + 1, c, d, e} += dp_{a, b, c, d, e} (b < n_2, a > b)dpa,b+1,c,d,e​+=dpa,b,c,d,e​(b<n2​,a>b)

# P2893 [USACO08FEB] Making the Grade G

似乎有无数倍经验。

本题有非常妙的贪心做法，这里就不讲了，来说一下如何 dp\text{dp}dp。

首先需要知道一个结论：

修改后的高度肯定都是输入中出现过的高度。

就不证明了。

那么我们就有一个比较显然的状态，设 dpi,jdp_{i, j}dpi,j​ 表示修改到第 iii 条路，修改后的高度是 hjh_jhj​，转移方程：

dpi,j=min⁡{dpi−1,k}+∣ai−hj∣(1≤k≤j)dp_{i, j} = \min\{dp_{i - 1, k} \} + |a_i - h_j|\ \ (1 \leq k \leq j)dpi,j​=min{dpi−1,k​}+∣ai​−hj​∣ (1≤k≤j)

aia_iai​ 是输入的第 iii 条路的高度，hih_ihi​ 是 aia_iai​ 排序去重之后的结果。

但是直接转移是 O(n3)O(n^3)O(n3) 的，考虑如何优化。

不难发现 min⁡{dpi−1,k}\min\{dp_{i - 1, k} \}min{dpi−1,k​} 这个东西非常多余，我们在转移当前层的时候再开个数组记一下最小值即可。

然后复杂度就变成 O(n2)O(n^2)O(n2) 的了。

# SP703 SERVICE - Mobile Service

看到这道题，先不要想太多，大胆设状态，把所有需要存下来的都存下来，于是有一个很暴力的状态，设 dpi,x,y,zdp_{i, x, y, z}dpi,x,y,z​ 表示处理到了第 iii 个询问，三个人分别在 x,y,zx, y, zx,y,z 三个位置上时的最小花费。

转移也很暴力：

dp[i+1][pi+1][y][z]=min⁡{dp[i][x][y][z]+w(x,pi+1)}dp[i+1][x][pi+1][z]=min⁡{dp[i][x][y][z]+w(y,pi+1)}dp[i+1][x][y][pi+1]=min⁡{dp[i][x][y][z]+w(z,pi+1)}dp[i + 1][p_{i + 1}][y][z] = \min\{dp[i][x][y][z] + w(x, p_{i + 1}) \} \\ dp[i + 1][x][p_{i + 1}][z] = \min\{dp[i][x][y][z] + w(y, p_{i + 1}) \} \\ dp[i + 1][x][y][p_{i + 1}] = \min\{dp[i][x][y][z] + w(z, p_{i + 1}) \} \\dp[i+1][pi+1​][y][z]=min{dp[i][x][y][z]+w(x,pi+1​)}dp[i+1][x][pi+1​][z]=min{dp[i][x][y][z]+w(y,pi+1​)}dp[i+1][x][y][pi+1​]=min{dp[i][x][y][z]+w(z,pi+1​)}

但是这样的时空复杂度我们无法接受，即使把第一维滚掉时间上依旧是 O(n3m)O(n^3m)O(n3m) 的。

如何优化呢？

我们观察到题目中时这么说的：

公司必须按照顺序，派一名员工到位置 xix_ixi​。

这又什么意义呢？相当于是告诉我们了在处理完第 iii 个请求时必有一个点在 pip_ipi​，所以我们只需要记录两个不在请求位置上的点的位置即可。

转移的话和上面几乎是一样的：

  • dp[i+1][x][y]=min⁡{dp[i][x][y]+w(pi,pi+1)}dp[i + 1][x][y] = \min\{dp[i][x][y] + w(p_i, p_{i + 1}) \}dp[i+1][x][y]=min{dp[i][x][y]+w(pi​,pi+1​)}，第三个人从 pip_ipi​ 跑到 pi+1p_{i + 1}pi+1​。
  • dp[i+1][x][pi]=min⁡{dp[i][x][y]+w(y,pi+1)}dp[i + 1][x][p_i] = \min\{dp[i][x][y] + w(y, p_{i + 1}) \}dp[i+1][x][pi​]=min{dp[i][x][y]+w(y,pi+1​)} 第二个人从 yyy 跑到 pi+1p_{i + 1}pi+1​，此时第三个人在上个请求中跑到了 pip_ipi​，所以把 pip_ipi​ 存下来。
  • dp[i+1][pi][y]=min⁡{dp[i][x][y]+w(x,pi+1)}dp[i + 1][p_i][y] = \min\{dp[i][x][y] + w(x, p_{i + 1}) \}dp[i+1][pi​][y]=min{dp[i][x][y]+w(x,pi+1​)} 同上面。

# P6064 [USACO05JAN]Naptime G

先假设不能构成环，考虑如何 dp\text{dp}dp。

设 dpi,j,0/1dp_{i, j, 0/1}dpi,j,0/1​ 表示到了第 iii 个时间，已经连着睡了 jjj 个时间，当前是否睡觉，0 表示不睡，1 表示睡。

那么显然有转移方程：

dpi,j,0=max⁡{dpi−1,j,1,dpi−1,j,0}dpi,j,1=max⁡{dpi−1,j−1,0,dpi−1,j−1,1+wi}dp_{i, j, 0} = \max\{dp_{i - 1, j, 1}, dp_{i - 1, j, 0} \} \\ dp_{i, j, 1} = \max\{dp_{i - 1, j - 1, 0}, dp_{i - 1, j - 1, 1} + w_i \}dpi,j,0​=max{dpi−1,j,1​,dpi−1,j,0​}dpi,j,1​=max{dpi−1,j−1,0​,dpi−1,j−1,1​+wi​}

但是时间是可以循环的，如何处理呢？

我们强制可以得到时间 1 的效用值即可，也就是强行把 1 和 nnn 连起来了。

这样为什么可以呢？

考虑如果我们没有得到时间 1 的效用值，那么即使时间可以循环也是无所谓的，所以这要就足够了。

我们只需要在赋初值时令 dp1,1,1=w1dp_{1, 1, 1} = w_1dp1,1,1​=w1​，且最后统计答案时只对 dpn,b,1dp_{n, b, 1}dpn,b,1​ 取较小值即可。

# P2687 [USACO4.3] 逢低吸纳 Buy Low, Buy Lower

O(n2)O(n^2)O(n2) 的最长下降子序列就不说了，直接来看方案数如何计算。

其实就是在最长下降子序列的 dpdpdp 数组上再跑一个 dp\text{dp}dp。

一般情况下，如果 dpi=dpj+1dp_{i} = dp_j + 1dpi​=dpj​+1 且 ai<aj(i>j)a_i < a_j\ \ (i > j)ai​<aj​ (i>j) 我们令 fi+=fjf_i += f_jfi​+=fj​，但是这样明显会算重很多东西。

（dpidp_idpi​ 为最长下降子序列的数组，fif_ifi​ 为计算方案数的数组）

考虑如何去重。

如果 dpi=dpjdp_i = dp_jdpi​=dpj​ 且 ai=aj(i>j)a_i = a_j\ \ (i > j)ai​=aj​ (i>j) 的话，那这个位置我们会多算，所以直接令 fj=0f_j = 0fj​=0 即可。

    fi,j,k=fi−1,fi−1,j,k,kdai,j,k=dai−1,j,k+dai−1,fi−1,j,k,kdbi,j,k=dbi−1,j,k+dbi−1,fi−1,j,k,kf_{i, j, k} = f_{i - 1, f_{i - 1, j, k}, k} \\ da_{i, j, k} = da_{i - 1, j, k} + da_{i - 1, f_{i - 1, j, k}, k} \\ db_{i, j, k} = db_{i - 1, j, k} + db_{i - 1, f_{i - 1, j, k}, k}fi,j,k​=fi−1,fi−1,j,k​,k​dai,j,k​=dai−1,j,k​+dai−1,fi−1,j,k​,k​dbi,j,k​=dbi−1,j,k​+dbi−1,fi−1,j,k​,k​

dpi,j=∑k=1i−1[ak<ai]dpk,j−1dp_{i, j} = \sum_{k = 1}^{i - 1}[a_k < a_i]dp_{k, j - 1}dpi,j​=k=1∑i−1​[ak​<ai​]dpk,j−1​

但是这个转移的复杂度是 O(n3)O(n^3)O(n3) 的，无法通过此题，考虑优化。

先不管 jjj 的一维，不难发现就是求 iii 前面小于 aia_iai​ 的和，直接整个树状数组优化即可。

# UVA1640 统计问题 The Counting Problem

数位 dp\text{dp}dp 板子题。

简单说一下吧，对于 0∼90 \sim 90∼9 分开考虑，即每次计算一个数字的出现次数。

数位 dp\text{dp}dp 基本思想，先计算 1∼r1 \sim r1∼r 的答案再减去 1∼l−11 \sim l - 11∼l−1 的答案。

假设当前要求 1∼x1 \sim x1∼x 中 now(now∈[0,9])now\ (now \in [0, 9])now (now∈[0,9]) 的出现次数。

还是 dfs(pos,cnt,lim,flag)dfs(pos, cnt, lim, flag)dfs(pos,cnt,lim,flag)：

  • pospospos：xxx 十进制下第几位。
  • cntcntcnt：nownownow 出现了多少次。
  • limlimlim：当前位是否有上限。
  • flagflagflag：前面是否都为前导 0。

里面的 dfsdfsdfs 调用时这么写：

dfs(pos−1,cnt+(!(flag&(!i))&(i==now)),lim&(i==tmp),flag&(!i))dfs(pos - 1, cnt + (!(flag \& (!i)) \& (i == now)), lim \& (i == tmp), flag \& (!i))dfs(pos−1,cnt+(!(flag&(!i))&(i==now)),lim&(i==tmp),flag&(!i))

其中 !(flag&(!i))!(flag \& (!i))!(flag&(!i)) 为判断当前位是否是前导 0，因为 nownownow 可能等于 0，同时 iii 也等于 0，如果此时是前导 0 的话不能统计到答案里面。

其他的就比较简单了，最后再开个 dp\text{dp}dp 数组记忆化一下。
